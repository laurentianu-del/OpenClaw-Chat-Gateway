# OpenClaw Chat Gateway v2.0.7

## 重点改进

- 网关设置中的浏览器“健康检查 / 自动修复”改为阶段式进度反馈：
  - 现在会按真实检查与修复步骤显示进度
  - 不再只是长时间等待或静态转圈
  - 三语文案已同步补齐 `zh-CN`、`zh-TW`、`en`

## 稳定性优化

- 浏览器自动修复的等待逻辑已收紧为：
  - 先做轻量就绪轮询
  - 浏览器运行态就绪后，再做一次完整官方链路验收
- 这样可以避免在轮询阶段反复执行 `open https://example.com` 和 `snapshot`，减少重复操作与等待时间。

## 交互优化

- 浏览器“自动修复”按钮现在默认置灰。
- 只有健康检查明确返回异常时，自动修复才会激活可点击，避免在未检查或检查失败类型不明确时误触发修复。

## 配置清理

- 网关设置中移除了遗留的 “OpenClaw 工作区路径” 配置项：
  - 页面不再显示该字段
  - 前后端不再读写该遗留配置
  - 不影响当前智能体与团队工作区的自动目录规则
